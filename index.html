🔧 Supabase Connection Setup
One-time configuration to connect to your Supabase project.
How to get your credentials:
  1. Log into supabase.com and open your project
  2. Project URL: Go to Settings → API → copy the Project URL
  3. Anon Key: Same page → copy the anon public key
  4. Paste both below — they are saved only in your browser.
Run this SQL in your Supabase SQL Editor (Dashboard → SQL Editor → New Query) to create the submissions table and storage bucket:
-- Create the submissions table CREATE TABLE IF NOT EXISTS submissions ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, ref_id TEXT, task TEXT, attempt INTEGER DEFAULT 1, tos_agreed BOOLEAN DEFAULT false, user_agent TEXT, photo_url TEXT, ip_address TEXT, flagged BOOLEAN DEFAULT false, notes TEXT, submitted_at TIMESTAMPTZ DEFAULT now(), created_at TIMESTAMPTZ DEFAULT now() ); -- Enable Row Level Security ALTER TABLE submissions ENABLE ROW LEVEL SECURITY; -- Policy: allow inserts from anon (for the public form) CREATE POLICY "Allow anonymous inserts" ON submissions FOR INSERT TO anon WITH CHECK (true); -- Policy: allow full access for authenticated users (admin) CREATE POLICY "Allow authenticated full access" ON submissions FOR ALL TO authenticated USING (true) WITH CHECK (true); -- Policy: allow anon to read (for admin via anon key) CREATE POLICY "Allow anon read" ON submissions FOR SELECT TO anon USING (true); -- Policy: allow anon to update (for flagging) CREATE POLICY "Allow anon update" ON submissions FOR UPDATE TO anon USING (true) WITH CHECK (true); -- Policy: allow anon to delete CREATE POLICY "Allow anon delete" ON submissions FOR DELETE TO anon USING (true); -- Create storage bucket for evidence photos INSERT INTO storage.buckets (id, name, public) VALUES ('evidence', 'evidence', true) ON CONFLICT (id) DO NOTHING; -- Allow public read access to evidence bucket CREATE POLICY "Public read evidence" ON storage.objects FOR SELECT TO public USING (bucket_id = 'evidence'); -- Allow anon uploads to evidence bucket CREATE POLICY "Anon upload evidence" ON storage.objects FOR INSERT TO anon WITH CHECK (bucket_id = 'evidence');
